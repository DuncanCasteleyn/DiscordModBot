# Privacy Policy

Last updated: 2026-06-11

This privacy policy describes how DiscordModBot processes data when it is used in a Discord server. DiscordModBot is
open-source software. If you self-host or modify the bot, you are responsible for your own deployment, database, Redis
instance, backups, logs, access controls, and any privacy notices required for your server or jurisdiction.

## Data We Process

DiscordModBot processes data needed to provide moderation, logging, role, reporting, voting, and member gate features.
Depending on which features are enabled in a server, this can include:

- Discord identifiers, such as guild IDs, user IDs, channel IDs, role IDs, message IDs, and emoji IDs.
- Server configuration, such as configured log channels, ignored channels, report channels, member gate channels,
  welcome channels, rules channels, role menus, sticky roles, trap channels, voting emotes, and related settings.
- Moderation records, such as warning points, warning reasons, moderator IDs, target user IDs, creation times,
  expiration times, scheduled unmutes, mute role data, trap-channel unban schedules, and sticky-role snapshots.
- Member gate data, such as configured questions, submitted answers, pending review prompts, reviewer IDs, pending
  user IDs, review session state, welcome message text, and welcome image URLs.
- Message logging data, such as recent message content, message author IDs, attachment URLs, emote text, message delete
  audit state, and message metadata used to report deleted or updated messages to configured log channels.
- Reporting and activity data, such as tracked role or member IDs and report channel IDs.
- Operational logs generated by the bot process, which may include Discord IDs or error context needed to diagnose bot
  behavior.

Discord itself may provide additional metadata to the bot through the Discord API and gateway. Discord's own collection
and processing of data is governed by Discord's policies.

## How We Use Data

DiscordModBot uses this data to:

- Respond to slash commands, buttons, modals, and other Discord interactions.
- Enforce moderation actions such as warnings, mutes, scheduled unmutes, trap-channel bans, member gate approval, and
  role restoration.
- Store server-specific configuration so features keep working after the bot restarts.
- Produce moderation and user-facing logs in the Discord channels configured by server administrators.
- Generate reports and summaries requested by configured bot features.
- Diagnose errors, prevent abuse, and maintain bot reliability.

The project does not sell personal data and does not use personal data for advertising.

## Where Data Is Stored

Persistent bot data is stored in the configured application database. Short-lived workflow data is stored in Redis when
Redis-backed features are used.

Some data is also sent back into Discord, for example moderation logs, message-delete logs, member gate review logs,
welcome messages, and command responses. Data sent to Discord is retained according to Discord's systems and the server'
s
own channel permissions, retention practices, and moderation choices.

## Retention

DiscordModBot keeps data only as long as needed for the enabled feature or until it is removed by an available reset,
delete, or configuration command.

- Recent message content, attachment URL caches, and message-delete audit state are kept in Redis for about 24 hours.
- Member gate modal questions are kept in Redis for about 10 minutes.
- Submitted member gate answers and review prompts are kept in Redis for about 48 hours.
- Member gate review sessions are kept in Redis for about 12 hours.
- Narou novel pending alert state is kept in Redis for about 15 minutes.
- Persistent server configuration remains until changed, reset, or deleted by server administrators or the bot operator.
- Moderation records remain for the lifetime required by the moderation feature, including configured expiration or
  scheduled action times when applicable.
- Discord messages created by the bot, including logs and transcripts in configured channels, remain in Discord until
  deleted by someone with the required Discord permissions or according to Discord/server retention rules.

Backups, host logs, and database snapshots are controlled by the bot operator or hosting environment and may have their
own retention periods.

## Sharing

DiscordModBot shares data only as needed to operate the bot:

- With Discord, because the bot receives events from Discord and sends command responses, moderation logs, and other bot
  messages through Discord.
- With the configured hosting, database, Redis, logging, and backup infrastructure used by the bot operator.
- With server administrators and moderators through configured log channels, review messages, reports, and command
  output.
- With external services only when a feature requires it. For example, the Narou novel alert feature uses public novel
  metadata and does not require Discord user data.

## Access And Deletion Requests

Server administrators can remove much of the server-specific configuration through the bot's reset and delete commands.
Users who want access to, correction of, or deletion of data associated with them should contact the relevant Discord
server administrators or the operator of the bot instance installed in that server.

For the public project repository, use GitHub to contact the project maintainer:
https://github.com/DuncanCasteleyn/DiscordModBot

If your request is about a security vulnerability, follow the process in `SECURITY.md` instead of opening a public
issue.

## Security

The bot should be deployed with access controls appropriate for the server and hosting environment. Bot tokens, database
credentials, Redis credentials, backups, logs, and Discord log channels should be protected from unauthorized access.

Report security vulnerabilities using the process in `SECURITY.md`.

## Changes

This policy may be updated when the bot's features or data processing change. Material changes should be reflected in
this file.
